# 跨学科头脑风暴方法论

> 为 STORM 模式提供：从七大学科框架库中系统扫描、嫁接和生成跨学科选题的操作体系。

---

## 一、三阶段 STORM 工作流

```
frame/ 框架加载 → 交叉扫描 (四维) → 理论嫁接 (五策略) → RQ 生成 → 评估精炼
```

### 阶段 1：多框架加载

根据 SKILL.md 路由表匹配的学科（≤3 个），完整 Read 对应的 `frame/` 文件，提取：

| 提取项 | 内容 |
|--------|------|
| **核心概念** | 每个理论的核心概念及其操作化定义 |
| **机制链条** | X→M→Y 的因果路径 |
| **边界条件** | 理论成立的前提和调节变量 |
| **竞争理论** | 与该理论预测相反或替代的理论 |
| **实证证据** | 关键实证发现和效应量 |

### 阶段 2：交叉扫描（四维）

#### 维度 1：概念同构扫描

在学科 A 和学科 B 的理论之间寻找共享或可通约的核心概念。

```
[学科A 理论X] 核心概念: [concept_A]
  定义: [definition]  机制: [mechanism]
[学科B 理论Y] 核心概念: [concept_B]
  定义: [definition]  机制: [mechanism]
同构分析: 共同点 / 差异点 / 统一概念 / 可导出的经验命题
```

#### 维度 2：机制互补扫描

寻找学科 A 的上游机制与学科 B 的下游机制之间的串联可能。

```
上游 (来自学科A): X → [mechanism_A] → M
下游 (来自学科B): M → [mechanism_B] → Y
整合检验策略: 分步检验 X→M, M→Y, 和完整间接效应 X→M→Y
```

#### 维度 3：理论冲突扫描

识别两个学科中对同一现象给出相反或不同预测的理论。

```
[学科A] 预测: [prediction_A] 前提: [assumptions_A]
[学科B] 预测: [prediction_B] 前提: [assumptions_B]
冲突诊断: 前提冲突 / 机制竞争 / 层次错位
调解方案: [调节变量假设] / [竞争中介假设] / [多层次整合]
```

#### 维度 4：方法迁移扫描

将一个学科成熟的分析方法应用于另一个学科的理论检验。

| 源学科 | 可迁移方法 | 目标学科 | 适用理论问题 |
|--------|-----------|---------|-------------|
| 经济学 | DiD / RD / IV | 社会学/教育学/公管 | 政策因果效应 |
| 心理学 | 实验范式 | 政治学/传播学 | 态度/行为因果 |
| 传播学 | 内容分析/NLP | 社会学/政治学 | 话语/框架分析 |
| 计算机科学 | 网络分析/ML | 全部学科 | 大规模关系/文本数据 |

## 三、学科框架快速索引（FRAME 模式用）

进行单学科 FRAME 探索时，从 `frame/` 中提取五维理论定位：

| 维度 | 关键问题 | frame 文件中的对应信息 |
|------|---------|----------------------|
| **机制定位** | 理论指定的因果链是什么？ | 每条理论的"机制链条" |
| **边界定位** | 理论在什么条件下成立/不成立？ | 每条理论的"边界条件" |
| **竞争定位** | 哪些理论提供替代/相反解释？ | 每条理论的"竞争/替代理论" |
| **实证定位** | 关键实证支持的强度如何？ | 每条理论的"实证证据" |
| **空白定位** | 哪些环节未被检验或存在矛盾？ | 综合"竞争理论"与"实证证据"推断 |

## 四、FATAL FLAW 识别清单（STORM 用）

在 RQ 生成后、评估前，每个 RQ 必须通过以下筛查。任一项触发 → 标记 FF，移出候选池：

- [ ] **概念不可操作化**：核心概念无法转化为可观测变量
- [ ] **同义反复**：X 和 Y 是同一构念的不同表述
- [ ] **数据不可及**：检验所需数据不存在且无法采集
- [ ] **理论嫁接断裂**：跨学科连接仅停留于表面类比（"A 就像 B"），未指定实质机制
- [ ] **层次错位**：宏观理论直接套用微观现象或反之，未说明跨层次机制
- [ ] **循环论证**：RQ 的答案已被其前提假设所决定
- [ ] **伦理不可行**：检验需要违反研究伦理
- [ ] **学科错配**：所选理论与研究主题的学科归属不一致

## 五、STORM 输出规范

每次 STORM 运行输出：

1. **多框架加载摘要**：涉及学科、加载理论数、关键概念清单
2. **交叉扫描矩阵**：四维扫描结果（概念同构/机制互补/理论冲突/方法迁移）
3. **理论嫁接方案**（3-5 个）：策略类型、源理论、嫁接命题、适配度评估
4. **候选 RQ 池**（15-20 个）：编号、陈述、嫁接来源、变量指定
5. **评估评分卡**：多维度评分 + FATAL FLAW 标记 + Rank
6. **Top 10 RQs**：精炼后的最终推荐，每个含理论锚点和检验策略
