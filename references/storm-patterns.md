## 1) 理论嫁接策略库

### 策略 A: 概念移植 (Concept Transplantation)

将一个学科的核心概念引入另一个学科的研究领域。

**操作步骤:**
1. 识别源学科的核心概念及其运作机制
2. 在目标学科中寻找同构的经验现象
3. 验证概念移植的"适配度" — 概念的前提条件在目标领域是否成立
4. 调整概念的边界条件以适应新领域
5. 推导可检验的经验命题

**经典案例:**
- Bourdieu 的"场域"(field) + "资本"(capital) 从社会学移植到平台劳动研究
- 经济学的"信号传递"(signaling) 从劳动经济学移植到教育社会学 (Spence 1973 → Collins 1979)
- 生物学的"生态位"(niche) 从生态学移植到组织社会学 (Hannan & Freeman 1977)

**适配度检验清单:**
- [ ] 源概念的前提条件在目标领域是否成立？
- [ ] 移植后概念的核心机制是否仍然运作？
- [ ] 目标领域是否有本土概念可以替代或需要整合？
- [ ] 移植后是否可以导出新颖的经验预测？

### 策略 B: 机制串联 (Mechanism Chaining)

将两个学科的因果机制串联成完整的因果链。

**操作步骤:**
1. 识别学科 A 中解释 X→M 的机制
2. 识别学科 B 中解释 M→Y 的机制
3. 验证 M (中间环节) 在两个学科中的操作化一致性
4. 构建完整因果链 X → (mech_A) → M → (mech_B) → Y
5. 推导可分步检验的经验命题

**经典案例:**
- 政策反馈 (政治学) → 组织适应 (公共管理) → 个体行为改变 (心理学)
- 媒体框架 (传播学) → 公众态度形成 (政治心理学) → 投票行为 (政治学)
- 教育扩张 (教育学) → 文凭贬值 (社会学) → 劳动市场分化 (经济学)

### 策略 C: 边界重划 (Boundary Redrawing)

用一个学科的实证发现限定另一个学科理论的边界条件。

**操作步骤:**
1. 确定学科 A 中被视为"普遍成立"的理论命题
2. 从学科 B 中寻找该命题可能不成立的情境条件
3. 识别关键的调节变量
4. 提出"理论A 在条件 C 下成立,在条件 C' 下不成立"的边界假说
5. 设计能够区分两种条件的经验检验

### 策略 D: 方法迁移 (Method Migration)

用一个学科成熟的分析方法检验另一个学科的理论命题。

**操作步骤:**
1. 识别学科 A 的理论命题中对因果识别的特定要求
2. 从学科 B 中寻找满足该要求的方法工具
3. 验证方法迁移的技术可行性 (数据格式、样本量、假设满足)
4. 说明方法迁移带来的因果推断优势
5. 制定方法迁移的实施计划

**常见迁移模式:**
- 经济学的 DiD/RD/IV → 社会学的教育/健康/犯罪研究
- 心理学的实验范式 → 政治学的态度/行为研究
- 传播学的内容分析 → 社会学的文化/话语研究
- 计算机科学的 NLP/网络分析 → 所有社会科学的大规模文本/关系数据

### 策略 E: 反事实挑战 (Counterfactual Challenge)

用一个学科的反直觉发现挑战另一个学科的共识性预设。

**操作步骤:**
1. 确定目标学科中广泛接受的"共识命题"
2. 从源学科中寻找与共识命题相反的反直觉发现
3. 检查反直觉发现的前提条件是否在目标领域存在
4. 构建竞争假设: H_target (共识) vs H_source (反直觉)
5. 设计能够区分两种假设的经验检验

## 2) 交叉扫描详细模板

### 2.1 概念同构扫描模板

```
概念同构扫描: [学科A] × [学科B]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[学科A 理论X] 核心概念: [concept_A1]
  定义: [definition]
  机制: [mechanism]

[学科B 理论Y] 核心概念: [concept_B1]
  定义: [definition]
  机制: [mechanism]

同构分析:
  共同点: [shared features]
  差异点: [key differences]
  统一概念: [synthesized concept]
  统一测量: [operationalization proposal]

可导出的经验命题:
  P1: [proposition 1]
  P2: [proposition 2]
```

### 2.2 机制互补扫描模板

```
机制互补扫描: [学科A] × [学科B]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

上游机制 (来自学科A):
  [theory_A]: [X] → [mechanism_A1] → [M]

下游机制 (来自学科B):
  [theory_B]: [M] → [mechanism_B1] → [Y]

整合检验策略:
  第一步: 检验 X → M (学科A的预测)
  第二步: 检验 M → Y (学科B的预测)
  第三步: 检验完整的间接效应 X → M → Y

潜在冲突:
  [如果 M 在两个理论中操作化不一致,如何协调?]
  [如果 M→Y 存在反向因果,如何解决?]
```

### 2.3 理论冲突扫描模板

```
理论冲突扫描: [学科A] × [学科B]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[学科A 理论] 预测: [prediction_A]
  前提: [assumptions_A]
  机制: [mechanism_A]

[学科B 理论] 预测: [prediction_B] (与A相反/不同)
  前提: [assumptions_B]
  机制: [mechanism_B]

冲突诊断:
  前提冲突: A和B的前提条件是否互斥？
  机制冲突: A和B的机制是否在同一因果路径上竞争？
  层次冲突: A和B的预测是否在不同分析层次上 (微观 vs 宏观)？

调解方案:
  [如果冲突源于前提差异 → 调节变量假设]
  [如果冲突源于机制竞争 → 竞争中介假设]
  [如果冲突源于层次差异 → 多层次整合模型]
```

## 3) 多智能体评估协议

### 3.1 评估准备

评估输入包 (通过 Agent 工具 prompt 传递):

```markdown
# 评估任务: 跨学科理论嫁接研究问题评估

## 研究主题
[用户的研究主题描述]

## 涉及的学科
[学科A] × [学科B] [× 学科C]

## 理论定位矩阵
[从 FRAME 模式 Step 3F 输出的前5个理论]

## 交叉扫描结果
[概念同构/机制互补/理论冲突/方法迁移 矩阵]

## 理论嫁接方案
[3-5个嫁接方案,各包含策略名称、源理论、嫁接命题]

## 候选研究问题 (15-20个)
[每个RQ: 编号、陈述、嫁接来源、变量指定]

## 评估要求
对每个候选 RQ 按6个维度打分 (1-10分):
1. 理论创新性 — 是否提供现有文献没有的新见解
2. 理论嫁接合理性 — 跨学科连接是否逻辑自洽
3. 可检验性 — 是否可用经验数据检验
4. 发表潜力 — 是否符合目标期刊偏好
5. 数据可行性 — 是否存在可及的数据来源
6. 社会意义 — 是否回应重要现实问题

识别:
- FATAL FLAW: 任何使RQ无法完成的根本性问题
- ★: 特别突出的强项
- 改进建议: 具体可操作的修改方案

最终给出每个 RQ 的总分 (加权平均) 和 Rank (1-20)
```

### 3.2 代理角色定义

**理论家 (Theorist) — 权重 30%**
- 关注: 概念创新性、理论逻辑自洽性、与现有文献的关系
- 标准: 是否是"有趣的"理论问题 (Davis 1971 的"有趣指数")
- 低分信号: 同义反复、概念拉伸 (conceptual stretching)、缺乏falsification条件
- 高分信号: 挑战共识、揭示隐秘联系、统一原先分离的解释

**方法论者 (Methodologist) — 权重 25%**
- 关注: 可检验性、识别策略、测量有效性
- 标准: 是否能够被经验研究严格检验
- 低分信号: 概念不可操作化、缺乏识别策略、混淆变量无法控制
- 高分信号: 清晰的数据来源、自然的识别变异 (natural experiment)、可复制

**领域专家 (Domain Expert) — 权重 20%**
- 关注: 文献定位准确性、与现有对话的衔接
- 标准: RQ 是否回应了该领域的核心争论
- 低分信号: 领域内已有大量文献回答此问题、与核心争论脱节
- 高分信号: 重新激活旧的未解争论、将边缘议题带入核心

**期刊编辑 (Editor) — 权重 15%**
- 关注: 发表潜力、读者群匹配度、对期刊的贡献
- 标准: 发表在目标期刊上的可能性
- 低分信号: 适用范围过窄 (niche)、无政策含义、方法争议过大
- 高分信号: 跨学科吸引力、清晰的政策/实践含义、方法"干净" (clean identification)

**魔鬼代言人 (Devil's Advocate) — 权重 10%**
- 关注: 致命缺陷、替代解释、反证可能性
- 标准: 能否经受住最严厉的学术批评
- 低分信号: 存在无法排除的替代解释、假设不可证伪、数据根本不存在
- 高分信号: 能被证伪但难以被证伪、对替代解释有预先设计

### 3.3 FATAL FLAW 判定标准

以下任一情况标记为 FATAL FLAW (FF):
- **概念不可操作化**: 核心概念无法转化为可观测变量
- **同义反复**: 解释变量与被解释变量是同一事物的不同表述
- **数据不可及**: 检验该 RQ 所需的数据目前不存在且无法采集
- **伦理不可行**: 检验该 RQ 需要违反研究伦理
- **逻辑矛盾**: RQ 内部存在逻辑矛盾 (如: 检验"X 对 Y 的因果效应"但 X 和 Y 同时决定)

### 3.4 代理派发

使用 Agent 工具并行派发 5 个代理:

```
Agent 1 (理论家): "你是社会学理论专家。评估以下15-20个候选RQ的理论创新性和逻辑自洽性..."
Agent 2 (方法论者): "你是定量/混合方法方法论专家。评估以下RQ的可检验性和识别策略..."
Agent 3 (领域专家): "你是[研究主题所属领域]的专家。评估以下RQ与现有文献对话的质量..."
Agent 4 (期刊编辑): "你是[目标期刊]的副主编。评估以下RQ的发表潜力..."
Agent 5 (魔鬼代言人): "你是批判性审稿人。找出以下每个RQ的致命缺陷和替代解释..."
```

## 4) RQ 评估标准与评分细则

### 4.1 六维度评分锚点

**理论创新性** (1-10)
- 1-3: 已知关系的重复/微小变体
- 4-6: 将已有理论应用于新情境
- 7-8: 提出新的机制或边界条件
- 9-10: 挑战现有共识,提出范式转换性的新见解

**理论嫁接合理性** (1-10)
- 1-3: 两个理论强行缝合,逻辑跳跃明显
- 4-6: 有合理的连接点但需要更多推演
- 7-8: 逻辑自洽,机制链条清晰
- 9-10: 嫁接后的框架比原理论更简洁或解释力更强

**可检验性** (1-10)
- 1-3: 概念模糊,无明确操作化路径
- 4-6: 可检验但需要大量辅助假设
- 7-8: 有明确的检验策略和变量操作化
- 9-10: 存在自然的识别变异,检验条件理想

**发表潜力** (1-10)
- 1-3: 领域内已饱和,或过于niche
- 4-6: 中等竞争力,适合领域期刊
- 7-8: 强竞争力,适合顶刊
- 9-10: 跨学科突破性,可能成为高引论文

**数据可行性** (1-10)
- 1-3: 数据根本不存在且采集成本极高
- 4-6: 存在相关数据但需大量处理
- 7-8: 公开数据可直接使用
- 9-10: 已有完美匹配的数据集,可立即开始分析

**社会意义** (1-10)
- 1-3: 纯学术兴趣,无实践含义
- 4-6: 有间接的政策/实践启示
- 7-8: 直接回应紧迫的社会问题
- 9-10: 研究结果可能影响政策制定或公共讨论

## 5) 常见跨学科陷阱与规避

### 陷阱 1: 概念拉伸 (Conceptual Stretching)
- 表现: 将一个学科的概念过度扩展,使其失去原有的分析精度
- 例: 将"资本"概念无限扩展到"社会资本""文化资本""符号资本""情感资本"...
- 规避: 每次概念移植前明确保留哪些维度、放弃哪些维度

### 陷阱 2: 隐喻式嫁接 (Metaphoric Grafting)
- 表现: 仅基于表面类比 (如"X就像Y") 进行理论连接,缺乏实质的机制共享
- 例: "城市就像一个生态系统" — 但未指定生态学机制的哪个方面适用于城市
- 规避: 要求每个嫁接方案都指定明确的因果机制,而非仅停留在隐喻层面

### 陷阱 3: 层次错位 (Level Mismatch)
- 表现: 将宏观层面的理论直接套用到微观层面的现象,或反之
- 例: 用国家层面的"制度同构"理论解释个体层面的组织行为
- 规避: 明确指定理论的分析层次,多层次研究需说明跨层次机制

### 陷阱 4: 术语通胀 (Term Inflation)
- 表现: 跨学科时引入过多术语,使理论框架臃肿难懂
- 规避: 优先使用最简洁的概念集合,非必要不引入新术语

### 陷阱 5: 单向掠夺 (One-Way Poaching)
- 表现: 仅从另一个学科中提取有利证据,忽视该学科内部的争论和不确定性
- 规避: 在引用跨学科理论时,必须同时引用该学科内部的竞争观点和局限性讨论

### 陷阱 6: 流行词嫁接 (Buzzword Grafting)
- 表现: 将时髦概念 (如"韧性""赋能""内卷") 强行附加到已有理论上,徒增噪音
- 规避: 每次引入新概念时必须论证其对理论解释力的"净增益"
