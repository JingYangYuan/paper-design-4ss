# STORM 模式：跨学科评估协议与嫁接策略库

---

## 一、理论嫁接五策略

### 策略 A：概念移植 (Concept Transplantation)

将一个学科的核心概念引入另一个学科，在目标领域寻找同构现象并验证适配度。

**操作步骤**：
1. 从源学科的 `frame/` 条目中提取概念的定义、机制和前提条件
2. 在目标学科中寻找与该概念机制同构的经验现象
3. 检验前提条件在目标领域是否成立（适配度检验清单）
4. 导出可检验的经验命题

**经典案例**：
- Bourdieu 的"场域"+"资本"从社会学 → 平台劳动研究
- 经济学的"信号传递"从劳动经济学 → 教育社会学 (Spence 1973 → 文凭信号)
- 生物学的"生态位"从生态学 → 组织社会学 (Hannan & Freeman 1977)

**适配度检验清单**：
- [ ] 源概念的前提条件在目标领域是否成立？
- [ ] 移植后概念的核心机制是否仍然运作？
- [ ] 目标领域是否有本土概念可以替代或需要整合？
- [ ] 移植后是否可以导出**新颖的**经验预测（而非仅用新术语重述已知发现）？

### 策略 B：机制串联 (Mechanism Chaining)

将两个学科的因果机制串联成完整的因果链，以 M（中间环节）为枢纽。

**操作步骤**：
1. 从学科 A 的 frame 理论中提取解释 X→M 的机制
2. 从学科 B 的 frame 理论中提取解释 M→Y 的机制
3. 验证 M 在两个学科中的操作化一致性
4. 构建 X → (mech_A) → M → (mech_B) → Y 的完整因果链
5. 设计可分步检验的经验命题

**经典案例**：
- 政策反馈（政治学）→ 组织适应（公共管理）→ 个体行为改变（心理学）
- 媒体框架（传播学）→ 公众态度形成（政治心理学）→ 投票行为（政治学）
- 教育扩张（教育学）→ 文凭贬值（社会学）→ 劳动市场分化（经济学）

**机制串联的适配检验**：
- [ ] M 在两个学科中的操作化定义是否一致？
- [ ] X→M 和 M→Y 的时间顺序是否可被数据支持？
- [ ] 串联后的因果链是否比单一学科的解释更完整或更简洁？

### 策略 C：边界重划 (Boundary Redrawing)

用一个学科的实证发现限定另一个学科理论的边界条件。

**操作步骤**：
1. 确定学科 A 中被视为"普遍成立"的理论命题（来自 frame 的"核心主张"）
2. 从学科 B 中寻找该命题可能不成立的情境条件（来自 frame 的"边界条件"）
3. 识别关键的调节变量 W
4. 提出"理论 A 在 W=w1 时成立，在 W=w2 时不成立"的边界假说
5. 设计能够区分两种条件的经验检验

**经典案例**：
- 社会认同理论的"内群体偏好"被系统正当化理论（心理学）限定边界——低地位群体可能表现出外群体偏好
- 效率工资理论（经济学）被公共部门特征（公共管理）限定——公共部门劳动力市场的工资-努力关系不同于私营部门

### 策略 D：方法迁移 (Method Migration)

用一个学科成熟的分析方法检验另一个学科的理论命题。

**操作步骤**：
1. 识别学科 A 的理论对因果识别的特定要求
2. 从学科 B 中寻找满足该要求的方法工具
3. 验证方法迁移的技术可行性（数据格式、样本量、假设满足）
4. 说明方法迁移带来的因果推断优势

**常见迁移模式**：

| 源学科 | 方法 | → 目标学科 | 典型应用 |
|--------|------|-----------|---------|
| 经济学 | DiD / RD / IV | 社会学/教育学/公管 | 政策因果效应评估 |
| 心理学 | 实验范式 | 政治学/传播学 | 态度形成与行为改变 |
| 传播学 | 内容分析 / NLP | 社会学/政治学 | 话语与框架分析 |
| 计算机科学 | 网络分析 / ML | 全部学科 | 大规模数据和关系结构 |

### 策略 E：反事实挑战 (Counterfactual Challenge)

用一个学科的反直觉发现挑战另一个学科的共识性预设。

**操作步骤**：
1. 确定目标学科中的"共识命题"（来自 frame 的"核心主张"）
2. 从源学科中寻找与该命题相反的反直觉发现（来自 frame 的"实证证据"中的反常发现）
3. 检查反直觉发现的前提条件是否在目标领域存在
4. 构建竞争假设：H_target（共识）vs. H_source（反直觉）
5. 设计能够区分两种假设的经验检验

## 二、交叉扫描模板

### 模板 1：概念同构扫描

```
概念同构扫描: [学科A: frame/theory-frameworks-X.md] × [学科B: frame/theory-frameworks-Y.md]

[理论X] 核心概念: [concept_A]
  定义: [来源: frame条目]
  机制: [来源: "机制链条"]
  前提条件: [来源: "边界条件"]

[理论Y] 核心概念: [concept_B]
  定义: [来源]
  机制: [来源]
  前提条件: [来源]

同构分析:
  · 共同点: [两者共享的核心逻辑]
  · 差异点: [关键分歧——概念范围/分析层次/前提假设]
  · 可统一概念: [synthesized concept]
  · 适配度评估: [前提条件在目标领域的成立情况]

可导出的经验命题:
  P1: [若统一概念成立，应观察到...]
  P2: [若差异点来自调节变量 W，则在 W 变动时...]
```

### 模板 2：机制互补扫描

```
机制互补扫描: [学科A] × [学科B]

上游机制 (来自学科A):
  [理论_A]: X → [mechanism_A1] → [mechanism_A2] → M

下游机制 (来自学科B):
  [理论_B]: M → [mechanism_B1] → [mechanism_B2] → Y

整合检验策略:
  第一步: 检验 X → M (验证学科A的预测)
  第二步: 检验 M → Y (验证学科B的预测)
  第三步: 检验完整间接效应 X → M → Y + 直接效应 X → Y 的残余

潜在冲突与协调:
  [如果 M 在两个理论中操作化不一致: 如何协调?]
  [如果 M→Y 存在反向因果: 如何解决?]
  [如果串联后解释力下降(而非上升): 是否说明学科A或B的机制指定有误?]
```

### 模板 3：理论冲突扫描

```
理论冲突扫描: [学科A] × [学科B]

[理论A] 预测: [prediction_A]
  前提: [assumptions_A]
  机制: [mechanism_A]

[理论B] 预测: [prediction_B] (与A相反/不同)
  前提: [assumptions_B]
  机制: [mechanism_B]

冲突诊断:
  · 前提冲突: A和B的前提条件是否互斥？→ 若互斥 → 调节变量假设 (W 决定哪个前提成立)
  · 机制竞争: A和B的机制是否在同一因果路径上竞争？→ 若竞争 → 竞争中介假设
  · 层次冲突: A和B的预测是否在不同分析层次上？→ 若不同层次 → 多层次整合模型

调解方案:
  [基于诊断结果的具体方案]
```

## 三、多智能体评估协议

### 3.1 评估维度与权重

| 维度 | 权重 | 评分标准 |
|------|------|---------|
| **理论创新性** | 20% | 1-3=已知重复; 4-6=新情境应用; 7-8=新机制/新边界; 9-10=挑战共识 |
| **理论嫁接合理性** | 20% | 1-3=强行缝合; 4-6=有连接点; 7-8=逻辑自洽、机制清晰; 9-10=嫁后框架解释力更强 |
| **可检验性** | 20% | 1-3=概念模糊; 4-6=需大量辅助假设; 7-8=有明确检验策略; 9-10=存在自然识别变异 |
| **数据可行性** | 15% | 1-3=数据不存在; 4-6=需大量处理; 7-8=公开数据可及; 9-10=完美匹配的数据集 |
| **发表潜力** | 15% | 1-3=已饱和/过于niche; 4-6=中等竞争力; 7-8=顶刊潜力; 9-10=跨学科突破性 |
| **实质意义** | 10% | 1-3=纯学术; 4-6=间接启示; 7-8=回应紧迫问题; 9-10=影响政策/公共讨论 |

### 3.2 FATAL FLAW 判定标准

标记 FF 并移出候选池：
- **概念不可操作化**：核心概念无法转化为可观测变量
- **同义反复**：X 和 Y 是同一构念的不同表述
- **数据不可及**：检验所需数据目前不存在且无法采集
- **伦理不可行**：检验违反研究伦理
- **嫁接断裂**：跨学科连接仅停留于表面类比（"A 就像 B"），未指定实质因果机制
- **逻辑矛盾**：RQ 内部存在逻辑矛盾

### 3.3 共识评分卡模板

```
| RQ# | 创新性(20%) | 嫁接性(20%) | 可检验(20%) | 数据(15%) | 发表(15%) | 意义(10%) | 加权总分 | Rank | 共识 |
|-----|------------|------------|------------|----------|----------|----------|---------|------|------|
| RQ1 |     8.5    |    7.0     |    8.0     |   7.5    |   6.0    |   7.0    |   7.48   |   3  |  ★★  |
| RQ2 |     9.0    |    8.5     |    9.0     |   9.0    |   8.0    |   8.0    |   8.65   |   1  | ★★★  |

共识标注: ★★★ = 5/5 维度 ≥ 8 | ★★ = 4/5 | ★ = 3/5 | — = <3/5 | FF = FATAL FLAW
```

## 四、常见跨学科陷阱与规避

| 陷阱 | 表现 | 规避 |
|------|------|------|
| **概念拉伸** | 将一个学科的概念过度扩展至失去分析精度（如"资本"无限扩展） | 每次概念移植前明确保留/放弃哪些维度 |
| **隐喻式嫁接** | 仅基于表面类比（"X 就像 Y"），缺乏机制共享 | 要求每个嫁接方案指定明确的因果机制，而非仅停留于隐喻 |
| **层次错位** | 宏观理论直接套用微观现象或反之 | 明确理论的分析层次；多层次研究需说明跨层次机制 |
| **术语通胀** | 跨学科时引入过多术语，框架臃肿 | 优先最简洁的概念集合，非必要不引入新术语 |
| **单向掠夺** | 仅从另一学科提取有利证据，忽视该学科内部的争论 | 引用跨学科理论时，必须同时引用竞争观点和局限性讨论 |
| **流行词嫁接** | 将时髦概念强行附加到已有理论上 | 每次引入新概念时论证其对理论解释力的"净增益" |
