# Phase 02: STORM 模式 — 跨学科头脑风暴

## 触发条件

- 用户明确指定 `storm` 模式 (默认模式)
- 用户关键词: `头脑风暴`, `头孢风暴`, `选题`, `跨学科`, `理论嫁接`, `创新选题`
- 用户在 FULL 模式的第二阶段
- 路由结果匹配到 2+ 个学科时自动建议

## 输入要求

| 字段 | 必需 | 说明 |
|------|------|------|
| 研究主题 | 是 | 具体的研究现象或话题 |
| 关联学科 | 是 | 2-3 个相关学科 (由路由层确定或用户指定) |
| 目标期刊 | 推荐 | 用于调整 RQ 评估的发表潜力维度 |
| 数据来源 | 推荐 | 用于评估 RQ 的可行性 |
| FRAME 报告 | 否 | 如有先前的 FRAME 输出，可作为起点 |

## 核心理念

"头孢风暴" = 将不同学科的理论框架视为"抗生素"，针对研究问题这个"病原体"，通过跨学科理论嫁接生成创新性的"治疗方案"。

## 执行流程

### Step 1: 多框架加载

加载 1-3 个相关学科的 frame 文件。对每个学科提取:

- 与研究主题相关的理论条目 (每学科 10-15 个)
- 每个理论的核心主张、机制链条、实证证据强度、理论地位
- 标注理论之间的"竞争/替代/互补"关系

**加载策略**: 先用 `grep` 扫描标题建立索引，再按相关度精读具体条目。

### Step 2: 跨学科交叉扫描 (Cross-Frame Scanning)

在两个或三个学科之间进行四维交叉扫描:

#### 扫描维度 1: 概念同构

识别不同学科用不同术语描述相同/相似社会过程的案例:

```
概念同构矩阵: [学科A] × [学科B]

[学科A理论X] 中的 [概念a]
    ≅ (等价于)
[学科B理论Y] 中的 [概念b]

共同指向: [更深层的社会机制]
可操作化为: [统一测量方案]
理论贡献: [揭示两个学科共享的深层逻辑]
```

**输出格式**:

| # | 概念A (学科A) | 概念B (学科B) | 同构逻辑 | 深层机制 | 可操作性 |
|---|-------------|-------------|---------|---------|---------|
| 1 | [概念a] | [概念b] | [为什么等价] | [共享机制] | ★★★★☆ |

#### 扫描维度 2: 机制互补

识别两个学科的理论机制可以串联形成完整因果链:

```
机制串联图:

[学科A理论] → [机制M1: X→Y] → [中间状态Y]
                                    ↓
[学科B理论] → [机制M2: Y→Z] → [最终结果Z]

完整因果链: X → M1 → Y → M2 → Z
可检验命题:
  H1: X 对 Y 有正向影响 (来自学科A)
  H2: Y 对 Z 有正向影响 (来自学科B)
  H3: Y 在 X→Z 关系中起中介作用 (跨学科机制串联)
```

**输出格式**:

| # | 机制A (学科A) | 机制B (学科B) | 串联逻辑 | 完整因果链 | 可检验性 |
|---|-------------|-------------|---------|-----------|---------|
| 1 | [X→Y] | [Y→Z] | [串联点Y] | [X→Y→Z] | ★★★★☆ |

#### 扫描维度 3: 理论冲突

识别两个学科对同一现象有相反预测:

```
理论冲突矩阵:

[学科A预测]: [Prediction_A — 当条件C1成立时]
[学科B预测]: [Prediction_B — 当条件C2成立时] (相反)

关键调节条件: [C 是什么？何时A成立、何时B成立？]
可检验命题: [竞争假设 H_A vs H_B]
理论贡献: [划定两个理论的各自适用边界]
```

**输出格式**:

| # | 预测A (学科A) | 预测B (学科B) | 冲突点 | 候选调节变量 | 竞争假设 |
|---|-------------|-------------|-------|------------|---------|
| 1 | [Pred_A] | [Pred_B] | [对立面] | [C] | [H_A vs H_B] |

#### 扫描维度 4: 方法迁移

识别学科A的研究方法可用于检验学科B的理论命题:

```
方法迁移方案:

[学科A] 使用的 [方法M] (通常用于研究 [现象P_A])
    ↓ 迁移
[学科B] 的理论命题 [命题Q] (通常用 [方法N] 检验)

迁移优势: [方法M 相对于方法N 的优势]
迁移挑战: [方法迁移需要的适配]
```

### Step 3: 理论嫁接 (Theory Grafting)

基于交叉扫描结果，生成理论嫁接方案:

#### 嫁接策略库

| 策略 | 操作 | 创新等级 | 风险 | 示例 |
|------|------|---------|------|------|
| **概念移植** | 将学科A的概念工具引入学科B的现象解释 | ★★★★ | 概念适用性存疑 | 将 Bourdieu 的"场域"引入平台劳动研究 |
| **机制串联** | 串联两个学科的机制形成完整因果链 | ★★★★★ | 机制衔接可能断裂 | 政策反馈(政治学) → 组织适应(公共管理) → 个体行为改变(心理学) |
| **边界重划** | 用学科B的发现限定学科A理论的边界条件 | ★★★ | 边界可能过于狭窄 | 用传播学"知识沟"限定教育学"在线学习平等"假设 |
| **方法迁移** | 用学科A的方法检验学科B的命题 | ★★★★ | 方法适用性需验证 | 用经济学DID检验社会学不平等命题 |
| **反事实挑战** | 用学科A的反直觉发现挑战学科B的共识 | ★★★★★ | 挑战可能不成立 | 用行为经济学"挤出效应"挑战公共管理"绩效工资"共识 |

#### 嫁接方案模板

对每个嫁接方案，输出:

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
嫁接方案 #[N]: [方案名称]
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

嫁接策略: [策略类型]
源理论A: [学科A — 理论X — 核心主张]
源理论B: [学科B — 理论Y — 核心主张]
嫁接逻辑: [A的概念/机制如何与B的概念/机制结合, ≤150字]

嫁接后的新理论命题:
  P1: [核心命题]
  P2: [衍生命题]
  P3: [边界命题]

创新性评估: ★★★★☆
  - 是否超越了单一学科的视角: [是/否 + 理由]
  - 是否提供了新机制/新概念: [是/否 + 理由]
  - 是否可与现有跨学科研究对话: [是/否 + 引用]

可检验性评估: ★★★★☆
  - 概念是否可操作化: [是/否 + 方案]
  - 数据是否可及: [是/否 + 来源]
  - 识别策略是否可行: [是/否 + 方法]

发表潜力:
  - 目标期刊匹配: [期刊名 — ★★★★☆]
  - 理论贡献类型: [新解释/新机制/新边界/新方法]
  - 潜在审稿人关切: [2-3个可能的质疑]
```

### Step 4: 研究问题生成

基于理论嫁接方案，使用六种策略生成 15-20 个候选 RQ:

#### 策略 A: 理论驱动

从嫁接理论的核心命题直接推导 RQ:

```
嫁接命题: [P1]
  → RQ: [核心命题的经验版本]
  → X: [自变量], Y: [因变量]
  → 贡献类型: 新解释
```

#### 策略 B: 机制驱动

从因果链中的关键机制环节推导 RQ:

```
嫁接机制链: [X → M1 → M2 → Y]
  → RQ: [M1 在什么条件下被激活？/ M1→M2 是否成立？]
  → 贡献类型: 新机制
```

#### 策略 C: 冲突驱动

从学科间的理论冲突推导 RQ:

```
理论冲突: [Pred_A] vs [Pred_B]
  → RQ: [在条件 C 下，哪个预测成立？]
  → 贡献类型: 新边界
```

#### 策略 D: 边界驱动

从理论的边界条件和调节变量推导 RQ:

```
边界条件: [理论X仅在条件C下成立]
  → RQ: [条件C是否确实调节X→Y的关系？]
  → 贡献类型: 新边界
```

#### 策略 E: 情境驱动

将理论应用于新情境 (中国/数字时代/危机时刻) 推导 RQ:

```
原情境: [理论在西方/传统时代/稳定的检验]
  → 新情境: [中国/数字化/危机]
  → RQ: [理论在新情境下是否仍然成立？需要怎样的修正？]
  → 贡献类型: 新情境拓展
```

#### 策略 F: 方法驱动

从可用数据/方法的独特优势推导 RQ:

```
可用数据: [CFPS/CGSS/自采数据] — 优势: [面板/IV/自然实验]
  → RQ: [哪些理论命题可以利用这个数据优势进行检验？]
  → 贡献类型: 新方法/新证据
```

#### 候选 RQ 格式

每个候选 RQ 必须包含:

```
RQ#[编号]: [清晰的研究问题陈述，1-2句话]

理论嫁接来源: [涉及学科 + 嫁接策略]
X (核心自变量): [变量名 + 测量方向]
Y (因变量): [变量名 + 测量方向]
M (中介机制): [如有]
W (调节变量): [如有]

生成策略: [A/B/C/D/E/F]
预期理论贡献: [新解释/新机制/新边界/新方法]
初步可行性: ★★★★☆
  - 数据: [可用来源 or 需采集]
  - 方法: [推荐识别策略]
  - 风险: [主要可行性风险]

创新性: ★★★★☆
发表潜力: ★★★★☆
```

**覆盖要求**: 15-20 个 RQ 必须覆盖至少 4 种生成策略。

### Step 5: 多智能体并行评估

加载 `references/storm-patterns.md` 中的评估协议。

派发 5 个 Agent 进行并行评估:

| Agent | 角色 | 评估角度 | 权重 | 评分维度 |
|-------|------|---------|------|---------|
| **理论家** | 社会学理论专家 | 理论创新性、逻辑自洽性、概念清晰度 | 30% | 创新性+嫁接合理性 |
| **方法论者** | 定量/混合方法专家 | 可检验性、识别策略可行性、数据可及性 | 25% | 可检验性+数据可行性 |
| **领域专家** | 研究主题所属领域专家 | 文献定位准确性、与现有对话的衔接 | 20% | 理论定位+文献贡献 |
| **期刊编辑** | 目标期刊审稿人视角 | 发表潜力、与期刊读者群的匹配度 | 15% | 发表潜力 |
| **魔鬼代言人** | 批判性审视 | 致命缺陷识别、替代解释、可证伪条件 | 10% | FATAL FLAW标记 |

#### Agent 输入包

通过 Agent 工具的 prompt 传递以下信息:
1. 研究主题和学科路由结果
2. 理论定位矩阵 (Top 5 理论)
3. 交叉扫描矩阵 (四维度)
4. 理论嫁接方案 (3-5 个详细方案)
5. 候选 RQs (15-20 个)
6. 评估标准和权重
7. 目标期刊信息 (如有)

#### 评估维度 (每个 RQ, 1-10 分)

| 维度 | 1-3 分 | 4-6 分 | 7-8 分 | 9-10 分 |
|------|--------|--------|--------|---------|
| 理论创新性 | 已有大量文献 | 有一定新意 | 明显创新 | 突破性创新 |
| 嫁接合理性 | 强行缝合 | 勉强合理 | 逻辑自洽 | 自然融合 |
| 可检验性 | 无法经验检验 | 检验困难 | 可以检验 | 精准可检验 |
| 发表潜力 | 不适合目标期刊 | 勉强适合 | 比较适合 | 高度匹配 |
| 数据可行性 | 数据不可及 | 可能可及 | 有明确来源 | 数据已有 |
| 社会意义 | 纯学术 | 有一定意义 | 回应重要问题 | 重大现实意义 |

#### Agent 并行派发

使用 Agent 工具并行派发 5 个 Agent:

```
Agent 1 (理论家):    "评估以下 15-20 个 RQ 的理论创新性和嫁接合理性..."
Agent 2 (方法论者):  "评估以下 15-20 个 RQ 的可检验性和识别策略..."
Agent 3 (领域专家):  "评估以下 15-20 个 RQ 的文献定位和领域贡献..."
Agent 4 (期刊编辑):  "评估以下 15-20 个 RQ 的发表潜力..."
Agent 5 (魔鬼代言人): "找出以下 15-20 个 RQ 的致命缺陷..."
```

#### 汇总输出

**多智能体共识评分卡**:

| RQ# | 理论家(30%) | 方法论者(25%) | 领域专家(20%) | 期刊编辑(15%) | 魔鬼代言人(10%) | 加权总分 | 共识度 |
|-----|------------|-------------|-------------|-------------|---------------|---------|--------|
| 1 | 8.5 | 7.0 | 8.0 | 7.5 | 6.0 | 7.55 | ★★★ |
| 2 | 9.0 | 8.5 | 8.5 | 9.0 | 8.0 | 8.65 | ★★★★ |
| ... | ... | ... | ... | ... | ... | ... | ... |

**共识度指标**:
- ★★★★★ = 5/5 代理给出 ≥8 分
- ★★★★ = 4/5 代理给出 ≥8 分
- ★★★ = 3/5 代理给出 ≥8 分
- ★★ = 2/5 代理给出 ≥8 分
- ★ = 1/5 代理给出 ≥8 分

### Step 6: RQ 精炼 (Revision)

对所有共识度 ≥★★★ 的 RQ，根据代理反馈进行精炼:

1. **回应对清晰度的批评**: 修改 RQ 措辞，增加操作化定义
2. **回应对可检验性的质疑**: 增加限定条件，缩小范围，或指定替代检验策略
3. **回应对"缝合感"的批评**: 强化理论嫁接逻辑，增加概念桥梁论证
4. **回应对数据可行性的质疑**: 指定替代数据来源或简化研究设计
5. **FATAL FLAW 处理**: 魔鬼代言人标记为 FATAL FLAW 的 RQ 移出 Top 10

输出原始 vs 精炼版对比表:

| RQ# | 原始版本 | 主要批评 | 精炼版本 | 改进说明 |
|-----|---------|---------|---------|---------|
| 1 | [原始RQ] | [批评摘要] | [精炼RQ] | [改进内容] |

### Step 7: 最终输出 — Top 10 研究问题

保存到 `output/paper-design/storm-report-[slug]-[date].md`

#### 报告结构

```
# STORM 报告: [研究主题]

## 1. 研究主题与多学科定位
- 研究主题: [完整描述]
- 涉及学科: [2-3 个学科]
- 路由依据: [关键词匹配逻辑]

## 2. 跨学科交叉扫描矩阵
- 概念同构: [N 组]
- 机制互补: [M 组]
- 理论冲突: [P 组]
- 方法迁移: [Q 组]

## 3. 理论嫁接方案
- 3-5 个详细嫁接方案
- 每个方案: 嫁接策略 + 双源理论 + 新命题 + 创新性/可检验性评估

## 4. 候选 RQs (15-20 个)
- 按生成策略分组 (A/B/C/D/E/F)
- 每个 RQ 附带: 理论来源 + 变量指定 + 预期贡献

## 5. 多智能体评估共识评分卡
- 加权总分排名
- 各代理分项分数
- 跨代理一致性分析

## 6. 精炼后的 Final Top 10 RQs
- 每个 RQ 附带:
  - 理论嫁接来源 (学科 + 理论 + 嫁接策略)
  - 变量指定 (X → M → Y → W)
  - 预期理论贡献类型
  - 可行性评估 (数据 + 方法 + 风险)
  - 发表潜力评估
  - 与原始版本的对比 (如有精炼)

## 7. 研究程序建议
- Quick-Win: 数据已有、方法成熟、风险较低的 RQ (建议优先启动)
- Medium-Investment: 需要数据采集或方法学习
- Deep-Investment: 需要全新数据采集或方法开发

## 8. 下一步建议
- 进入 DESIGN 模式将选定 RQ 转化为研究设计蓝图
- 或参考 references/ 中的方法论资料进行进一步打磨
```

## STORM 模式质量检查

完成前自检:
- [ ] 交叉扫描覆盖至少两个学科，四维度至少完成两类
- [ ] 理论嫁接方案明确标注嫁接策略类型 (概念移植/机制串联/边界重划/方法迁移/反事实挑战)
- [ ] 每个嫁接方案有创新性+可检验性双维度评估
- [ ] 15-20 个候选 RQs 覆盖至少 4 种生成策略 (A/B/C/D/E/F)
- [ ] 每个候选 RQ 包含: 理论嫁接来源 + 变量指定 + 预期贡献
- [ ] 多智能体评估全部 5 个代理并行派发
- [ ] 共识评分卡包含跨代理一致性分析
- [ ] FATAL FLAW RQ 被标记并移出 Top 10
- [ ] 精炼版 RQ 附有原始对比和改进说明
- [ ] 每个 Top 10 RQ 包含完整的变量指定和理论嫁接来源
