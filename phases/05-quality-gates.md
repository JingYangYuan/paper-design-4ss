# Phase 05: 质量门控与输出规范

## 质量门控体系

paper-design-4ss 采用分层质量门控，每层有不同的检查标准和阻断规则。

### 门控层次

```
Layer 0: 路由质量 (入口门控)
    ↓ 通过
Layer 1: FRAME 模式质量
    ↓ 通过
Layer 2: STORM 模式质量
    ↓ 通过
Layer 3: DESIGN 模式质量
    ↓ 通过
Layer 4: 整体质量 (出口门控)
```

---

## Layer 0: 路由质量 (入口门控)

**检查时机**: 学科路由完成后、模式分发前

**阻断规则**: 路由错误 → 中断，要求用户确认学科归属

### 检查清单

| # | 检查项 | 标准 | 未通过处理 |
|---|--------|------|-----------|
| L0-1 | 学科路由正确 | 主题关键词匹配到正确的 frame 文件 | 要求用户确认或手动指定学科 |
| L0-2 | 多学科组合合理 | 学科之间互补而非冗余 (如社会学+经济学 优于 社会学+公共管理) | 调整学科组合，给出组合理由 |
| L0-3 | 路由理由明确 | 路由决策有可追溯的关键词匹配记录 | 补充匹配日志 |
| L0-4 | 模式选择合理 | 模式与用户需求匹配 (有RQ→DESIGN, 无RQ→STORM/FRAME) | 向用户确认模式选择 |

### 学科冗余度检查

两个学科被认为"冗余"当它们共享 ≥60% 的核心理论视角:

| 学科对 | 冗余度 | 建议 |
|--------|--------|------|
| 社会学—公共管理 | 中 (40%) | 可组合，但需明确分工 |
| 公共管理—政治学 | 高 (65%) | 通常选其一，除非有明确跨学科需求 |
| 心理学—传播学 | 低 (25%) | 互补性好，推荐组合 |
| 社会学—经济学 | 低 (20%) | 互补性好，推荐组合 |
| 教育学—心理学 | 中 (45%) | 可组合，注意微观-微观重叠 |

## Layer 1: FRAME 模式质量

**检查时机**: FRAME 报告生成后

**阻断规则**: 理论定位矩阵 < 3 个候选理论 → 扩大搜索范围或切换学科

### 检查清单

| # | 检查项 | 标准 | 阻断 |
|---|--------|------|------|
| L1-1 | Frame 文件完整加载 | 所有理论板块均被扫描 | 否 |
| L1-2 | 理论定位矩阵充足 | ≥ 5 个候选理论 | 是 (< 3) |
| L1-3 | 匹配度理由充分 | 每个理论有完整的五维评估 | 否 |
| L1-4 | 空白识别有理论依据 | 空白清单引用 frame 文件中的信息 | 否 |
| L1-5 | 四类空白覆盖 | 至少覆盖两类空白 (理论内部/理论间/经验/方法论) | 否 |
| L1-6 | RQ 方向有理论根基 | 推荐方向可追溯到具体理论 | 是 (无追溯) |

---

## Layer 2: STORM 模式质量

**检查时机**: STORM 报告生成后、多智能体评估完成后

**阻断规则**: 
- 交叉扫描未覆盖至少 2 个学科 → 补充学科
- FATAL FLAW RQ 未清理 → 清理后重新排序
- Top 10 中有 RQ 共识度 < ★★ → 降级或移除

### 检查清单

| # | 检查项 | 标准 | 阻断 |
|---|--------|------|------|
| L2-1 | 交叉扫描覆盖学科 | ≥ 2 个学科 | 是 |
| L2-2 | 扫描维度覆盖 | 四维度 (概念同构/机制互补/理论冲突/方法迁移) 至少完成两类 | 否 |
| L2-3 | 嫁接策略标注 | 每个嫁接方案标注策略类型 (概念移植/机制串联/边界重划/方法迁移/反事实挑战) | 否 |
| L2-4 | 双维度评估 | 每个嫁接方案有创新性+可检验性评估 | 是 |
| L2-5 | RQ 策略覆盖 | 15-20 个 RQ 覆盖 ≥ 4 种生成策略 | 是 |
| L2-6 | RQ 信息完整 | 每个 RQ 包含: 理论嫁接来源 + 变量指定 + 预期贡献 | 是 |
| L2-7 | 多智能体评估 | 全部 5 个代理并行派发 | 是 |
| L2-8 | 共识分析 | 评分卡包含跨代理一致性分析 | 否 |
| L2-9 | FATAL FLAW 清理 | FATAL FLAW RQ 被标记并移出 Top 10 | 是 |
| L2-10 | Top 10 完整 | 每个 Top 10 RQ 包含完整的变量指定和理论嫁接来源 | 是 |

### FATAL FLAW 判定标准

魔鬼代言人标记为 FATAL FLAW 的条件 (满足任一):

| 标记 | 条件 | 示例 |
|------|------|------|
| 🔴 逻辑矛盾 | RQ 的前提与结论逻辑冲突 | "检验A→B" 但理论预测的是 B→A |
| 🔴 套套逻辑 | RQ 的结论已经包含在前提中 | "理性人会做理性选择" |
| 🔴 不可检验 | 核心概念无任何可操作化路径 | "社会潜意识对政策偏好的影响" |
| 🔴 数据不存在 | 所需数据在可预见未来不存在 | 需要某特定人群的百年追踪数据 |
| 🔴 伦理不可行 | 研究设计违反研究伦理 | 需要随机分配贫困状态 |
| 🟡 缝合感过强 | 跨学科连接仅为表面类比，无实质机制 | "教育就像市场经济" 但没有具体机制 |

## Layer 3: DESIGN 模式质量

**检查时机**: DESIGN 报告生成后

**阻断规则**: 理论-方法对齐检查有 "不通过" 项 → 修正后重新检查

### 检查清单

| # | 检查项 | 标准 | 阻断 |
|---|--------|------|------|
| L3-1 | 理论锚点已确认 | 理论框架 + 机制链条 + 变量结构清晰 | 是 |
| L3-2 | 变量操作化有理论依据 | 每个测量指标追溯到理论概念 | 否 |
| L3-3 | 识别策略与数据匹配 | 策略适合数据结构 (面板→FE/DID, 截面+IV→2SLS) | 是 |
| L3-4 | 功效分析已执行 | 定量设计包含功效分析或样本量论证 | 否 |
| L3-5 | 理论-方法对齐 | 七项检查全部 "通过" 或 "需调整" | 是 (任何 "不通过") |
| L3-6 | 竞争假设设计 | 设计能区分理论预测与竞争理论预测 | 否 |
| L3-7 | 稳健性方案充足 | 覆盖 ≥ 4 种威胁 | 否 |

---

## Layer 4: 整体质量 (出口门控)

**检查时机**: 最终输出前 (任何模式)

**阻断规则**: 四项核心标准中任一项不满足 → 返回相应阶段修正

### 核心标准

| # | 标准 | 评估方法 | 阻断 |
|---|------|---------|------|
| Q1 | **跨学科连接有实质内容** | 理论嫁接方案中的机制链条是否可追溯？是否仅为表面类比？ | 是 |
| Q2 | **理论嫁接逻辑自洽** | 从源理论 → 嫁接命题 → RQ 的逻辑链是否完整无断裂？ | 是 |
| Q3 | **研究方向可操作** | 推荐的 RQ/设计方向是否可以用经验方法检验？ | 是 |
| Q4 | **输出文件完整** | 所有必需的报告文件是否已保存？ | 是 |

### 表面类比检测

以下模式视为"表面类比"而非实质理论嫁接:

- ✗ "A就像B一样" — 没有具体机制
- ✗ "A领域的X概念对应B领域的Y概念" — 仅概念对应，无机制连接
- ✗ "借鉴A学科的视角看B问题" — 模糊，无具体理论操作
- ✓ "A理论中的X机制 (具体说明) 可以解释B领域中的Y现象 (具体映射)" — 实质机制

## 输出规范

### 文件命名规范

```
[项目根目录]/output/paper-design/
├── frame-report-[discipline]-[slug]-[YYYY-MM-DD].md
├── storm-report-[slug]-[YYYY-MM-DD].md
├── design-report-[slug]-[YYYY-MM-DD].md
└── full-report-[slug]-[YYYY-MM-DD].md
```

**slug 规则**: 研究主题关键词的英文缩写，≤20 字符，小写，用 `-` 连接
**date**: 生成日期，格式 YYYY-MM-DD

### Process Log 规范

```
[项目根目录]/output/logs/
└── process-log-paper-design-4ss-[YYYY-MM-DD].md
```

Process log 格式:

```markdown

- **Arguments**: [raw arguments]
- **Working Directory**: [pwd]
- **Mode**: [frame|storm|design|full]
- **Disciplines**: [matched disciplines]
- **Target Journal**: [if specified]

## Steps

| # | Timestamp | Step | Action | Output | Duration | Status |
```

### 报告最小内容标准

任何报告缺失以下标记为 `[REQUIRED]` 的章节 → 质量门控阻断:

**FRAME 报告**:
- [REQUIRED] 研究主题与学科定位
- [REQUIRED] 理论定位矩阵 (≥ 5 行)
- [REQUIRED] 研究空白清单 (≥ 5 行)

**STORM 报告**:
- [REQUIRED] 跨学科交叉扫描矩阵
- [REQUIRED] 理论嫁接方案 (≥ 3 个)
- [REQUIRED] 多智能体评估共识评分卡
- [REQUIRED] Final Top 10 RQs (每个含变量指定)

**DESIGN 报告**:
- [REQUIRED] 变量操作化表
- [REQUIRED] 实证模型设定
- [REQUIRED] 理论-方法对齐检查

**FULL 报告**:
- [REQUIRED] 所有三个阶段的报告
- [REQUIRED] 执行路线图

## 常见质量失败与修正

| 失败模式 | 检测方式 | 修正方案 |
|---------|---------|---------|
| 学科路由错误 | L0-1 失败 | 重新提取关键词，或让用户手动指定 |
| 理论定位肤浅 | L1-2 候选理论 < 5 个 | 扩大搜索范围，降低关键词匹配阈值 |
| 跨学科缝合感 | Q1 表面类比检测失败 | 回到 STORM Step 3，重新设计嫁接逻辑 |
| RQ 不可检验 | L2-5 魔鬼代言人 FATAL FLAW | 缩小 RQ 范围，指定替代数据来源 |
| 方法不匹配数据 | L3-3 失败 | 调整识别策略以匹配实际数据结构 |
| 本体论错配 | L3-5 "不通过" | 切换方法范式或更换理论框架 |
| 输出不完整 | Q4 失败 | 补充缺失章节 |
