# Phase 01: FRAME 模式 — 单学科理论框架探索

## 触发条件

- 用户明确指定 `frame` 模式
- 用户关键词: `理论框架`, `找理论`, `理论定位`, `理论锚点`
- 用户在 FULL 模式的第一阶段
- 路由结果仅匹配到单一学科时自动建议

## 输入要求

| 字段 | 必需 | 说明 |
|------|------|------|
| 研究主题 | 是 | 具体的研究现象或话题 |
| 主学科 | 是 | 由路由层确定，或用户指定 |
| 目标期刊 | 否 | 用于调整理论定位的深度和风格 |
| 已有理论偏好 | 否 | 用户是否已有倾向的理论视角 |

## 执行流程

### Step 1: 学科路由确认

运行路由规则，确定主要学科。若用户已指定学科则直接确认；若未指定则根据关键词自动匹配。

输出路由结果卡片:

```
╔══════════════════════════════════════════════════════════════╗
║  ROUTING RESULT                                               ║
╠══════════════════════════════════════════════════════════════╣
║  研究主题: [用户主题]                                          ║
║  主学科:   [sociological / public-admin / psychology / ...]    ║
║  备选学科: [如有]                                               ║
║  Frame文件: frame/theory-frameworks-[discipline].md            ║
╚══════════════════════════════════════════════════════════════╝
```

### Step 2: 加载理论框架

使用 Read 工具加载对应的 frame 文件。两步加载策略:

1. **快速扫描**: 读取 frame 文件前 200 行，提取 `##` 一级标题，建立理论条目索引
2. **按需精读**: 根据研究主题关键词，精读相关度最高的 5-8 个理论条目的完整内容

提取命令:

```bash
FRAME_FILE="${SKILL_DIR:-paper-design-4ss}/frame/theory-frameworks-${DISCIPLINE}.md"
grep "^## " "$FRAME_FILE" | head -50
```

呈现理论框架总览表:

```
[学科名称] 理论框架总览
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
一、[理论板块1] — [板块核心议题]
  1.1 [理论名] — [核心主张概括，≤30字]
  1.2 [理论名] — ...
  1.3 [理论名] — ...

二、[理论板块2] — [板块核心议题]
  2.1 [理论名] — ...
  ...

统计: [N] 个理论板块，[M] 个理论条目
```

### Step 3: 理论定位 (Theory Anchoring)

将用户研究主题与框架中的理论进行多维度匹配。对每个高度相关的理论，从五个维度评估:

#### 3.1 匹配度评估 (1-5★)

| ★ 数 | 标准 |
|-------|------|
| ★★★★★ | 理论核心命题直接解释研究现象，概念工具可直接操作化 |
| ★★★★☆ | 理论提供有力分析视角，但需微调概念框架 |
| ★★★☆☆ | 理论部分覆盖研究现象，需补充其他理论 |
| ★★☆☆☆ | 理论边缘相关，可作为辅助视角 |
| ★☆☆☆☆ | 理论基本不相关，强行套用会有"缝合感" |

#### 3.2 解释潜力评估

- **概念工具箱**: 该理论提供了哪些可用于分析的概念？
- **机制链条**: 该理论是否可以拆解出清晰的因果机制？
- **分析层次**: 该理论最适合分析哪个层次 (个体/群体/组织/制度/国家)？

#### 3.3 机制映射

将理论的抽象机制链条映射到用户研究现象的具体过程:

```
理论机制: [抽象因果链 A → B → C → D]
    ↓ 映射
经验过程: [用户现象中的 X → Y → Z → W]
    ↓
可检验命题: H1: X→Y, H2: Y→Z, H3: Z→W
```

#### 3.4 边界条件

从 frame 文件中提取该理论的适用边界:
- **适用层次**: 微观/中观/宏观
- **适用情境**: 稳定/变迁/危机
- **适用社会类型**: 西方/非西方/中国特定
- **已知局限性**: 理论已有的批判和局限

#### 3.5 输出: 理论定位矩阵

| # | 理论 | 板块 | 匹配度 | 解释潜力 | 机制可用性 | 边界条件 | 推荐等级 |
|---|------|------|--------|---------|-----------|---------|---------|
| 1 | [理论名] | [板块] | ★★★★★ | [视角, ≤20字] | [机制链, ≤30字] | [局限, ≤20字] | ★ 首选 |
| 2 | [理论名] | [板块] | ★★★★☆ | ... | ... | ... | 备选 |
| 3 | ... | ... | ... | ... | ... | ... | 备选 |
| ... | ... | ... | ... | ... | ... | ... | ... |

**至少输出 5 个候选理论**，前 2-3 个为首选推荐。

### Step 4: 研究空白识别 (Gap Identification)

基于 frame 文件中的"竞争/替代理论"、"理论地位"、"未解决的问题"信息，系统识别四类空白:

#### 4.1 理论内部空白

该理论自身内部尚未充分检验的环节:
- 核心命题是否全部经过严格实证检验？
- 机制链条中是否有"黑箱"环节？
- 理论的微观基础是否清晰？

#### 4.2 理论间空白

两个竞争理论之间的未解决张力:
- 理论A vs 理论B 在哪个预测上对立？
- 是否有研究同时检验两者的预测？
- 调节条件是什么 (何时A成立，何时B成立)？

#### 4.3 经验空白

理论预测缺乏实证的领域:
- **人群空白**: 理论是否在特定人群中检验过 (如中国/农村/老年人)？
- **情境空白**: 理论是否在特定情境中检验过 (如数字时代/危机时刻)？
- **时间空白**: 是否有纵向追踪检验？
- **空间空白**: 是否有跨国/跨地区比较？

#### 4.4 方法论空白

现有研究的方法局限:
- 因果识别不足 (缺乏外生冲击/DID/IV/RDD)
- 缺乏质性深描 (只有问卷，缺乏过程追踪)
- 缺乏混合方法 (定量与质性脱节)
- 样本局限性 (学生样本/便利样本/单一城市)

#### 4.5 输出: 研究空白清单

| # | 空白类型 | 具体描述 | 理论依据 | 可研究性 | 创新潜力 | 推荐方法 |
|---|---------|---------|---------|---------|---------|---------|
| 1 | 理论间空白 | [A vs B 在 X 上的对立] | [引用两个理论] | ★★★★☆ | ★★★★★ | [推荐方法] |
| 2 | 经验空白 | [理论在 Y 人群的缺失] | [引用理论] | ★★★☆☆ | ★★★★☆ | [推荐方法] |
| 3 | 方法论空白 | [现有研究缺乏 Z 方法] | [引用理论] | ★★★☆☆ | ★★★★★ | [推荐方法] |
| ... | ... | ... | ... | ... | ... | ... |

**至少识别 5 个空白**，按创新潜力排序。

### Step 5: 输出理论定位报告

保存到 `output/paper-design/frame-report-[discipline]-[slug]-[date].md`

#### 报告结构

```
# FRAME 报告: [研究主题]

## 1. 研究主题与学科定位
- 研究主题: [完整描述]
- 主学科: [学科名称]
- 路由依据: [关键词匹配逻辑]
- 备选学科: [如有]

## 2. 理论框架总览
- [学科] 理论框架全景图
- [N] 个理论板块，[M] 个理论条目

## 3. 理论定位矩阵
- 前 5 个最相关理论的详细五维分析
- 每个理论附带: 匹配度、解释潜力、机制映射、边界条件

## 4. 研究空白清单
- 按类型分类 (理论内部/理论间/经验/方法论)
- 每个空白附带: 理论依据、可研究性、创新潜力、推荐方法

## 5. 推荐研究方向
- 3-5 个具体研究方向
- 每个方向附带: 理论根基、预期贡献、初步可行性

## 6. 下一步建议
- 进入 STORM 模式进行跨学科拓展
- 或进入 DESIGN 模式直接设计方案
```

### Step 6: 日志记录

追加到 process log:

```bash
echo "| [step#] | $(date +%H:%M:%S) | FRAME-Complete | [discipline] frame report saved | frame-report-[discipline]-[slug]-[date].md | ✓ |" >> "$LOG_FILE"
```

## FRAME 模式质量检查

完成前自检:
- [ ] Frame 文件完整加载 — 所有理论板块均被扫描
- [ ] 理论定位矩阵至少包含 5 个候选理论
- [ ] 每个推荐理论都有完整的五维评估 (匹配度+解释潜力+机制映射+边界条件)
- [ ] 研究空白有明确的理论依据 (引用 frame 中的"竞争/替代理论"和"理论地位"信息)
- [ ] 四类空白 (理论内部/理论间/经验/方法论) 至少覆盖两类
- [ ] 推荐的 RQ 方向有理论根基而非随意发挥
- [ ] 报告文件已保存并包含所有六个必需章节
