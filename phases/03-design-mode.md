# Phase 03: DESIGN 模式 — 理论 → 研究设计蓝图

## 触发条件

- 用户明确指定 `design` 模式
- 用户关键词: `研究设计`, `方法设计`, `设计方案`, `设计蓝图`
- 用户已从 FRAME/STORM 获得理论锚点和 RQ
- 用户在 FULL 模式的第三阶段

## 输入要求

| 字段 | 必需 | 说明 |
|------|------|------|
| 研究问题 (RQ) | 是 | 从 FRAME/STORM 输出中选定的核心 RQ，或用户直接提供 |
| 理论锚点 | 是 | 核心理论框架及其机制链条 |
| 变量初步指定 | 是 | X → M → Y 的结构 |
| 数据来源 | 推荐 | CFPS/CGSS/CHARLS/自采数据/无 |
| 目标期刊 | 推荐 | 决定方法部分的格式和深度 |
| 方法偏好 | 否 | 定量/质性/混合方法 |

## 输入获取优先级

1. 从先前的 FRAME/STORM 报告中提取 (如果在同一会话中)
2. 要求用户指定 (如果直接进入 DESIGN 模式)
3. 如果用户跳过 FRAME/STORM 直接进入 DESIGN → 先运行 FRAME Step 1-3 确定理论锚点

## 执行流程

### Step 1: 理论锚点确认

确认用户在 FRAME 或 STORM 模式中确定的理论框架和 RQs。

#### 确认清单

```

研究问题: [核心 RQ，一句话]

  - M (中介): [变量名 / 操作化方向]
  - Y (因变量): [变量名 / 操作化方向]
  - W (调节): [变量名 / 操作化方向] (如有)
```

若用户直接进入 DESIGN 模式 (跳过 FRAME/STORM)，则先运行 Phase 01 的 Step 1-3 确定理论锚点。

### Step 2: 研究设计生成

#### 路线 A: 委托 scholar-design (推荐)

将以下信息打包传递给 scholar-design:

1. **研究问题**: 从 STORM/FRAME 输出中选定的核心 RQ
2. **理论锚点**: 核心理论框架及其机制链条
3. **变量初步指定**: X → M → Y 的结构
4. **可用数据**: 用户指定的数据来源或需要重新采集
5. **目标期刊**: 决定方法部分的格式和深度

提示用户使用 `/scholar-design [quant|qual|mixed] "[RQ]" [data source] [journal]`

#### 路线 B: 自主设计 (备选)

如果无法调用 scholar-design，在当前会话中执行以下设计步骤:

##### B.1 研究范式选择

| 研究目标 | 推荐范式 | 核心方法 |
|---------|---------|---------|
| 因果效应估计 | 定量—准实验 | DID, IV, RDD, PSM, FE |
| 机制检验 | 定量—中介/调节 | SEM, 因果中介分析, 调节效应 |
| 过程理解 | 质性—过程追踪 | 深度访谈, 参与观察, 档案分析 |
| 意义建构 | 质性—解释主义 | 叙事分析, 现象学, 扎根理论 |
| 三角验证 | 混合方法 | 定量+质性 并行/序贯设计 |

##### B.2 识别策略选择

根据数据结构选择识别策略:

| 数据结构 | 推荐策略 | 所需假设 |
|---------|---------|---------|
| 面板数据 (2+ 期) | FE / DID / 事件研究 | 平行趋势 / 严格外生 |
| 横截面 + IV | 2SLS / IV Probit | 相关 + 排他性 |
| 断点 | RDD (Sharp/Fuzzy) | 断点处随机 |
| 重复横截面 | 队列DID / 年龄-时期-队列 | 队列可比性 |
| 质性数据 | 过程追踪 / 比较案例分析 | 理论饱和度 |

##### B.3 变量操作化

从理论概念到经验测量:

| 理论概念 | 操作化变量 | 数据来源 | 测量方式 | 信效度证据 |
|---------|-----------|---------|---------|-----------|
| [概念X] | [变量名] | [数据集/问题编号] | [量表/指标] | [Cronbach α / CFA] |
| [概念Y] | [变量名] | [数据集/问题编号] | [量表/指标] | [Cronbach α / CFA] |
| ... | ... | ... | ... | ... |

##### B.4 功效分析

```bash
# 使用 Stata 或 R 进行功效分析
# 如需定量设计，推荐运行 power 命令
```

功效分析要素:
- 预期效应量 (基于先前文献或 pilot)
- 所需样本量
- 显著性水平 (α = 0.05)
- 统计功效 (1-β ≥ 0.80)

##### B.5 稳健性方案

| 威胁 | 稳健性检验 |
|------|-----------|
| 遗漏变量 | Oster bounds / 系数稳定性检验 |
| 测量误差 | 替代测量 / 潜变量模型 |
| 样本选择 | Heckman / 倾向得分匹配 |
| 反向因果 | 滞后自变量 / IV |
| 竞争理论 | 竞争假设检验 / 包含 rival 预测变量 |

### Step 3: 理论-方法一致性检查

在研究设计完成后，进行理论-方法对齐检查:

#### 对齐检查矩阵

| 检查项 | 评估标准 | 通过 | 需调整 | 不通过 |
|--------|---------|------|--------|--------|
| **本体论对齐** | 理论的认识论立场 (实证主义/解释主义/批判主义) 是否与方法匹配？ | [ ] | [ ] | [ ] |
| **机制检验** | 设计是否能够检验理论中指定的因果机制 (而不仅仅是 X→Y 关联)？ | [ ] | [ ] | [ ] |
| **边界条件** | 设计是否覆盖了理论的边界条件和调节变量？ | [ ] | [ ] | [ ] |
| **竞争假设** | 设计是否能够区分理论预测与竞争理论的预测？ | [ ] | [ ] | [ ] |
| **发表范式** | 设计是否符合目标期刊的方法论偏好和发表范式？ | [ ] | [ ] | [ ] |
| **层次匹配** | 理论和测量的分析层次是否一致 (个体/群体/组织/国家)？ | [ ] | [ ] | [ ] |
| **时间维度** | 设计的时间框架是否匹配理论的时间假设？ | [ ] | [ ] | [ ] |

#### 常见对齐问题与修正

| 问题 | 症状 | 修正方案 |
|------|------|---------|
| 本体论错配 | 用定量的因果推断检验解释主义的理论 | 调整为质性方法 / 或更换理论框架 |
| 机制黑箱 | 设计只检验 X→Y，未涉及中介机制 | 增加中介变量测量和中介分析 |
| 边界遗漏 | 未包含理论的边界条件变量 | 增加调节变量，检验交互效应 |
| 竞争理论混淆 | 无法区分理论A和理论B的预测 | 增加 rival 变量，设计竞争假设检验 |
| 层次谬误 | 用个体数据推断组织层次的理论 | 改用多层次模型 / 或调整理论层次 |

### Step 4: 输出研究设计蓝图

保存到 `output/paper-design/design-report-[slug]-[date].md`

#### 报告结构

```
# DESIGN 报告: [研究主题]

## 1. 理论锚点
- 研究问题: [核心 RQ]
- 理论框架: [学科 + 理论 + 核心主张]
- 机制链条: [X → M1 → M2 → Y]
- 理论贡献类型: [新解释/新机制/新边界/新方法]

## 2. 研究设计概览
- 研究范式: [定量/质性/混合]
- 设计类型: [实验/准实验/观察研究/案例研究/...]
- 识别策略: [DID/IV/RDD/过程追踪/...]
- 数据来源: [数据集/自采方案]

## 3. 变量操作化
- 变量操作化表 (概念 → 变量 → 测量 → 信效度)

## 4. 实证模型
- 基准模型设定
- 机制检验模型
- 异质性分析模型
- 稳健性检验方案

## 5. 功效分析
- 预期效应量 + 所需样本量 + 功效计算

## 6. 理论-方法对齐检查
- 七项检查结果
- 修正建议 (如有)

## 7. 预分析计划 (PAP) 概要
- 假设列表
- 模型设定
- 多重检验校正
- 排除/包含标准

## 8. 下一步
- 论文写作建议
- 预注册建议
- 数据采集计划 (如需)
```
